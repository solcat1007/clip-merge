# ClipMerge — 片段合并器

> 使用 ffmpeg concat demuxer 流拷贝模式实现多段同格式视频的无损合并，无需重新编码，画质零损失，速度仅受磁盘 I/O 限制。

## 功能列表

- **无损合并**：使用 ffmpeg `concat` demuxer + `-c copy` 流拷贝模式，不重新编码，画质/音质完全保留
- **格式一致性检测**：添加文件时自动检测格式是否一致，不一致时弹出警告（无损合并要求同格式）
- **顺序调整**：支持上移/下移按钮调整视频片段的合并顺序
- **进度反馈**：合并过程中显示不确定进度条，完成后弹窗显示输出文件路径和大小
- **自动命名**：输出文件以 `merged_时间戳.原格式` 命名，存放于第一个视频所在目录
- **临时文件清理**：合并完成后自动删除 concat 文件列表临时文件

## 安装方法

### 环境要求

- Python 3.8+
- ffmpeg（需自行安装并添加到系统 PATH）

### 安装 ffmpeg（Windows）

1. 从 [ffmpeg 官网](https://ffmpeg.org/download.html) 或 [gyan.dev](https://www.gyan.dev/ffmpeg/builds/) 下载 `ffmpeg-release-full.7z`
2. 解压到 `C:\ffmpeg\`，确保 `C:\ffmpeg\bin\ffmpeg.exe` 存在
3. 将 `C:\ffmpeg\bin` 添加到系统 PATH 环境变量

### 获取项目

```bash
git clone git@github.com:solcat1007/clip-merge.git
cd clip-merge
```

无需安装额外 Python 依赖，仅使用 Python 标准库。

## 详细使用

### 启动

```bash
python clip_merge.py
```

### 操作步骤

1. 点击 **「添加视频」** 选择需要合并的视频文件（建议同格式）
2. 使用 **「上移」「下移」** 调整合并顺序，或 **「移除选中」** 删除不需要的片段
3. 确认列表正确后点击 **「开始无损合并」**
4. 等待合并完成，弹窗显示输出路径和文件大小

### 使用示例

**场景：合并三个航拍片段**

```
1. 添加文件：航拍_01.mp4、航拍_02.mp4、航拍_03.mp4
2. 确认顺序：航拍_01 → 航拍_02 → 航拍_03
3. 点击「开始无损合并」
4. 输出：D:\素材\merged_20260708_143022.mp4
```

**场景：格式不一致时的处理**

```
若列表中包含 .mp4 和 .mov 文件，
→ 系统提示格式不一致，合并按钮禁用
→ 建议先用 FormatShift 等工具将文件统一为同一格式后再合并
```

## 输入输出说明

### 支持的输入格式

| 容器格式 | 扩展名 |
|---|---|
| MP4 | `.mp4` |
| QuickTime | `.mov` |
| AVI | `.avi` |
| Matroska | `.mkv` |
| Windows Media | `.wmv` |
| Flash Video | `.flv` |
| WebM | `.webm` |
| iTunes Video | `.m4v` |
| MPEG Transport Stream | `.ts` / `.mts` / `.m2ts` |

> ⚠️ 无损合并要求**所有输入文件格式完全相同**（包括编码参数）。若格式不一致，请先用转码工具统一。

### 输出

- 输出路径：**第一个视频文件所在目录**
- 文件名：`merged_YYYYMMDD_HHMMSS.原格式扩展名`（如 `merged_20260708_143022.mp4`）
- 若目标文件已存在，弹窗询问是否覆盖

### 合并原理

```
ffmpeg -f concat -safe 0 -i concat_list.txt -c copy -map 0 output.mp4
```

其中 `concat_list.txt` 内容格式：

```
file 'D:\素材\视频1.mp4'
file 'D:\素材\视频2.mp4'
file 'D:\素材\视频3.mp4'
```

`-c copy` 表示流拷贝，不经过编解码器，直接将原始数据流拼接，效果等同于二进制拼接但正确处理了容器元数据。

## 注意事项

1. 无损合并要求所有片段的编码参数完全一致（分辨率、帧率、编码格式、采样率等）。若参数不同，请先用转码工具统一参数后再合并
2. 合并速度受磁盘读写速度限制，与文件大小成正比。多个大文件在同一块机械硬盘上时速度会较慢——这是 I/O 瓶颈，非编码问题
3. 不支持无损合并不同格式的视频（如 MP4 + MOV）。工具会检测格式一致性，不一致时合并按钮禁用
4. 合并时 ffmpeg 报错 "unsupported codec"：某些视频编码可能不被 concat demuxer 支持。可尝试先将所有片段转码为统一格式（如 H.264 + AAC in MP4）后再用本工具合并
5. 输出文件比源文件总和稍大或稍小是正常的，concat demuxer 在合并时会重新生成容器元数据，画质和音质不受影响
